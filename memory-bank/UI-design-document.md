# Kotoba 桌面端 UI 设计方案

## 设计目标
以最少步骤和无压的提示，让用户在桌面端轻松完成造卡与复习，始终获得即时、温柔且不出错的反馈。

## 信息架构与交互流
- 导航：左侧窄栏仅保留「新增单词」「复习」「词库」「统计」四个入口，主区域单栏承载当前任务，避免多层跳转。
- 新增单词（极简 3 步）：1）聚焦输入框键入/粘贴单词并回车触发生成；2）AI 自动填充读音、释义、场景、例句并形成预览卡片，字段可立即编辑且自动保存草稿；3）「保存卡片」一键写入并提示「已加入今日计划」，可选自动跳转到复习队列。异常时保留用户输入并提供「手动完成」。
- 复习（极简 3 步）：1）点击「开始今日复习」自动加载待复习队列；2）显示卡片正面，空格或按钮翻面，顶部进度条同步更新；3）「容易/一般/困难」大按钮评分后即时更新 SM-2 并自动切到下一张。若队列为空，直接提示「今日无计划复习」并提供「自选复习」入口。
- 词库管理（极简 3 步）：1）进入列表默认按最近更新排序并显示搜索框；2）输入关键字或选择「仅今日」「按下次复习日」等筛选，列表即时过滤；3）点击行打开侧滑/对话框编辑，字段失焦即自动保存，删除需二次确认。
- 统计与活跃度（极简 3 步）：1）进入即见今日概览（新增数、复习数、连续天数）与「去复习」主按钮；2）下方展示活跃度热力格与词库难度占比，悬浮显示具体数值；3）点击任一数据点跳转到对应列表或复习队列。所有图表加载显示进度骨架。
- 默认值与自动化：新增页默认光标在输入框，回车触发生成；草稿自动保存；SM-2 更新后自动写入 JSON 并更新今日活跃度；导航记忆上次停留页；错误重试 1 次后提供手动路径。
- 关键动作即时反馈：按钮点击后 200ms 内展示 loading 状态，操作完成显示正向 toast，失败给出原因与「重试」。

## 界面布局与组件层级
- 总体：左窄栏导航 + 居中单栏内容区（宽 960px），顶部保留页面级标题与主行动按钮，首屏集中最主要操作。
- 新增单词：首屏为大号输入框与主按钮「生成卡片」，下方预览卡片区域显示自动填充内容，旁边提供轻量文本框编辑，底部固定主按钮「保存卡片」与次按钮「手动完成」。
- 复习：中央卡片（大字号 term/kana，翻面动画 200ms），下方评分按钮三等分，顶部显示进度条与「剩余 n 张」文案，右上有「结束本轮」次按钮。
- 词库管理：单栏列表/表格，表头固定搜索与筛选，列表行突出 term/kana，次级信息淡化；行右侧「编辑」「删除」为图标按钮，编辑浮层靠右出现，确保主列表可见。
- 统计：首屏三块数卡（新增/复习/连续天数），主 CTA「去复习」置于同一行；下方单列展示热力格与扇形图，留足间距避免拥挤。
- 可达性：按钮高度 ≥ 40px，点击区域充足；键盘操作支持 Tab/Enter/Space 全路径；主要颜色对比度遵循 WCAG AA。

## 视觉与动效设计
- 色彩：高对比低饱和方案，背景米白(#F8F5F0) 与浅灰(#EEF1F4)，主色深蓝绿(#1F6F8B) 用于主按钮与高亮，强调色暖橙(#F0A35E) 用于进度与提示。
- 字体：采用「Inter + Noto Sans JP」组合，标题加粗，正文常规；数字信息使用等宽「JetBrains Mono」强化可读性。
- 间距与密度：单栏布局保持 24px 内边距，组件垂直间距 16px，表格行高 ≥ 48px；卡片内元素按「标题/主信息/次信息」递减对比。
- 动效：翻面与进度变更使用 200ms ease-out；加载用骨架闪烁或细线进度，避免旋转等待；状态变更淡入淡出 150ms，不打断操作。
- 关键路径突出：首屏主按钮使用实心主色，次按钮描边浅色；副文本与标签使用次级灰，形成明确层级；图标统一线性风格。

## 交互文案样例
- 加载/等待：生成卡片时「正在为你补全读音和例句… 稍等片刻」，复习加载「整理今日卡片中…」
- 成功反馈：保存卡片「已保存，已加入今日计划」，复习评分「记忆计划已更新，下一张来了」
- 空状态：复习无计划「今日无需复习，想自选练习吗？」；词库为空「还没有单词，先添加一个吧」
- 错误与重试：生成失败「网络有点慢，改为手动填写或重试一次」，保存失败「写入失败，数据已保留，点此重试」
- 按钮文案：新增主按钮「生成卡片」/「保存卡片」，复习评分「容易」「一般」「困难」，结束「稍后再复习」，统计 CTA「去复习」
- 安抚语气：长等待时「先休息一下，我们正在处理你的卡片」，重试后「已帮你准备好再次尝试」
